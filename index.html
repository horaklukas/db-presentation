Databázový modeler na bázi Node.js

Horák Lukáš

k-ai2

Vedoucí práce: Ing. Pavel Kříž Ph.D.

Úvod

  • Modelování jako důležitá součást SW návrhu
  • Motivace k vytvoření webového modeleru => sdílená služba
  • Praktická část práce: vytvoření modeleru
  • Teoretická část: vývoj, použité SW nástroje, potřebné znalosti z oblasti relačních databází

Cílová aplikace

  • Navržena s jednoduchým ovládáním
  • Webová aplikace = více způsobů použití
  • Možnosti příjímání různých vstupů, produkování výstupů

Existující řešení

  • Různé parametry pro porovnání (cena, vlastnosti, platforma)
  • Pro konkrétní RDBS (MySQL Workbench)
  • Komplexní nástroje pro modelování (Enterprise Architect)
  • Online modelery (Vertabelo)

Technologie

  • Základem je platforma Node.js
  • Komunikace pomocí WebSockets
  • Closure Tools od Google
  • Knihovna React.js od Facebooku

Node.JS

  • Výkonná serverová platforma s asynchronní I/O modelem
  • Základ pro mnoho vývojových nástrojů (Mocha, Stylus,..)
  • Jednotný jazyk pro klienta i server - modularita a sdílení
  • NPM - nativní správce modulů

CoffeeScript a Stylus

  • Javascriptový překladač a CSS preprocesor
  • Řeší některé nedostatky starších standardů (ECMAScript5, CSS2)
  • Psaní méně "upovídaného" kódu = lepší čitelnost + menší zdrojové soubory

Externí knihovny

  • Google Closure tools = sada nástrojů (překladač, knihovna, linter)
  • Closure Library - dependency managment + spousta funkcionality (DOM, ...)
  • React.JS - malé grafické komponenty, rychlost aktualizace

Výsledná aplikace

  • SPA - komunikace pomocí AJAX, Websockets
  • Vlastní kód + klientské a serverové závislosti (NPM, Bower)
  • Export modelu i s aplikací do jednoho HTML souboru
  • Verzování modelů databázového schematu

Serverová část

  • Funkce webového serveru (Express.js)
  • Správa verzí modelů na souborovém systému
  • Třídy pro reverse engineering z různých RDBS
  • Poskytování definic cílových RDBS

Klientská část

  • Nekomplikované GUI (pracovní plocha, nástrojová lišta, dialogy), technologie HTML + CSS + SVG
  • Provázanost objektů v modelu pro interaktivní změny
  • Speciální mód - exportovaná aplikace - ukládání do LocalStorage

Zhodnocení výsledků

  • Výsledná aplikace dostupná v cloudu Heroku i na Githubu
  • Vyřešeny problémy vzniklé při vývoji (tabulka - vztah, výchozí schema)
  • Poznatky a návrhy z testování ve firmě Retia

Závěr

  • Výsledkem použitelná, dále se rozvíjející, aplikace
  • Nasazení do reálného provozu
  • Znalosti z oblasti relačních databází - způsob uložení informací o DB a objektech v katalogu
